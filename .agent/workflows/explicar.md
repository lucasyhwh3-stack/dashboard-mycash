# Workflow: Explicar Código

## Objetivo
Explicar o código de forma clara, didática e acessível para um desenvolvedor intermediário.

## Etapas

1. **Visão geral**
   - Qual o propósito deste código?
   - Onde ele se encaixa no sistema maior?
   - Que problema ele resolve?

2. **Estrutura e organização**
   - Como o código está organizado?
   - Quais são as partes principais?
   - Como elas se relacionam?

3. **Explicação linha a linha** (para trechos complexos)
   - Detalhe o que cada parte faz
   - Explique o "porquê", não apenas o "o quê"
   - Use analogias quando apropriado
   - Aponte patterns e práticas utilizadas

4. **Conceitos importantes**
   - Explique conceitos que podem ser novos
   - React hooks, padrões de design, etc.
   - Forneça contexto adicional quando necessário

5. **Fluxo de execução**
   - Como os dados fluem através do código?
   - Que eventos ou ações acionam o código?
   - Qual a ordem de execução?

6. **Decisões de design**
   - Por que foi implementado desta forma?
   - Quais alternativas existem?
   - Quais trade-offs foram feitos?

7. **Possíveis melhorias**
   - O que poderia ser melhorado?
   - Sugestões para aprendizado adicional
   - Recursos para entender melhor
