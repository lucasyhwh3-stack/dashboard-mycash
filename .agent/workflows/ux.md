# Workflow: Avaliação de UX

## Objetivo
Avaliar a funcionalidade sob a perspectiva de experiência do usuário e produto.

## Etapas

1. **Entenda o contexto**
   - Qual o objetivo do usuário ao usar esta funcionalidade?
   - Que problema ela resolve?
   - Quem é o usuário-alvo?

2. **Usabilidade**
   - A interface é intuitiva?
   - O fluxo é claro e lógico?
   - Há feedback visual adequado?
   - Estados de loading, erro e sucesso estão claros?
   - É acessível (cores, contraste, touch targets)?

3. **Experiência mobile**
   - Touch targets têm tamanho adequado (≥44x44px)?
   - Inputs têm altura mínima de 48px?
   - Font-size mínimo de 16px (evita zoom no iOS)?
   - Layout responsivo funciona bem?
   - Gestos são naturais?

4. **Performance percebida**
   - Sente-se rápido para o usuário?
   - Há estados intermediários (loading, skeleton)?
   - Feedback imediato nas ações?
   - Transições suaves?

5. **Mensagens e comunicação**
   - Mensagens de erro são claras e acionáveis?
   - Linguagem é amigável e não técnica?
   - Ajuda contextual quando necessário?

6. **Edge cases e prevenção de erros**
   - Como se comporta com dados vazios?
   - E se o usuário errar?
   - Há validações preventivas?
   - Confirmações para ações destrutivas?

7. **Recomendações**
   - Liste melhorias prioritárias
   - Sugira ajustes de UX
   - Identifique possíveis frustrações
   - Proponha melhorias incrementais
